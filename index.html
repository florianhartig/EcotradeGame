FAQ

Technichal problems

  • The applet does not load, or does not work correctly - Make sure that you have a current version (1.5 or higher) of Java installed. The latest version can be downloaded free of charge here. Also, make sure that you have not blocked java in the security settings of your browser. Try to empty your browser cache.

  • No Ecotrade server under the given address - If your applet displays this message, there are two possible explanations:

    • A firewall is blocking the connection between server and client. This is particularly likely when you try to connect to our server from the internet, but may also be the case on a local network. Make sure that either all firewalls between you and the server are turned off, or that the neccesary ports are openend. The game needs five ports for communication with the clients. These ports are the five consecutive ports starting from the port chosen in "server settings" (default 50000). Remember that additionally to your personal firewall on your computer, your company/institute may secure your internal network with additional firewalls which may limit access to these ports. In doubt, check with your IT staff.
    • There is no server running under the IP address you specified, or the port opened by the server and the port specified in your clients don't match. This is unlikely to be the case when you try to play the game on our website, but if you run the game on your local computer it may be that you have forgotten to start the server or specified wrong ports / IP addresses. Check whether you have started the server and updated the port information as described below.

  • How do I start the game on a local Network or on a single computer? - First, start a server on one of the computers in the network. Do this by either opening the .jar file without options and click on server or master view, or by executing java -jar ecotrade.jar -server. Then, start the ecotrade clients on the other computers either through opening the .jar file and clicking on client or typing java -jar ecotrade.jar -client. If you run the game on a local network, you have to make the ip address of the server known to the clients under "server settings". You will also be asked for the port. Default value for the port used by the sever is is 50000, you can change this with the option -port when you start the server. Now, everything should work. If the clients don't find the server, check below for firewall problems.

  • I have a technical problem which could not be solved here - Please contact the maintainer

General questions

*Why did you create the game? - EcoTRADE - The Game was created by Martin Drechsler, Florian Hartig and Martin Horn for the Wissenschaftssommer 2008 as a part of the year of mathematics in Germany. The programming and design was done by Martin Horn. The aim of the Wissenschaftssommer is to communicate scientific research to the public and in particular to raise young peoples interest for scientific projects and research.

*Who contributed to the development? Martin Drechsler, Florian Hartig, Martin Horn, Paul Vincent Kuper

*I am planning to use the software e.g. for an experiment and need additional features, or I would like to contribute to the development. Please contact Martin Drechsler and Florian Hartig
